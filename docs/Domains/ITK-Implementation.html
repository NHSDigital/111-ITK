Trigger Events

Send NHS 111 Report (POCD_TE000003GB01)

The NHS 111 Send trigger event indicates that a NHS 111 CDA document is ready to send from the originating system.

NHS 111 Nullify By Replacement (POCD_TE000004GB01)

For the NHS 111 CDA Document to be nullified the trigger event is the realisation that the NHS 111 CDA Document was sent in error, in that it contained an error and should never have been sent and that no correct NHS 111 CDA Document is available to send.

Send NHS 111 Ambulance Request (REPC_TE000001GB01)

The NHS 111 Ambulance Request trigger event indicates that an ambulance request or an update to an existing ambulance reuest is sent.

NHS 111 Repeat Caller Query (QUPA_TE000001GB01)

The NHS 111 Repeat Caller Query trigger event indicates that a query of the repeat caller system is triggered.

NHS 111 Repeat Caller Response (PRPA_TE000001GB01)

The NHS 111 Repeat Caller Response trigger event indicates that a response from the repeat caller system is triggered.

Repeat Caller Identified (PRPA_TE000012GB01)

The NHS 111 Repeat Caller Identified trigger event indicates that a caller has been identified as a repeat caller.

Primary General Practitioner Recipient NHS 111 CDA Document (urn:nhs-itk:interaction:primaryGeneralPractitionerRecipientNHS111CDADocument-v2-0)  

Communicates NHS 111 Report to a General Practitioner recipient where the recipient is recommended to act upon information carried in the document.

Note: This interaction has a message format that is constrained using HSCIC templates. This format is validated using a two level process. There are two sets of schema one for each level. The sender of this interaction must build all instances to the "on the wire" format using the generic CDA definition indicated in the wire format below. These instances must also conform to the rules stated in the conformance definition indicated in the conformance format below. The second level of validation requires transformation before schema validation can take place. For full details of this validation process please see the "Technical Guidance for Templated CDA Domains" section under the associated specifications tab.


Sender: NHS 111 Call Handler System (POCD_AR000004GB01)
Receiver: NHS 111 General Practitioner Receiving System (POCD_AR000005GB01)
Message:
  • Wire Format: NHS CDA RMIM (POCD_MT000002UK01)
  • Conformance Format: NHS111 Report (POCD_MT200001GB02)
Trigger Event: Send NHS 111 Report (POCD_TE000003GB01)
Transmission Wrapper: ITK Distribution Envelope (Version 2.0) (distributionEnvelope-v2-0)

Primary NHS 111 Repeat Caller Document Set (urn:nhs-itk:interaction:primaryNHS111RepeatCallerDocumentSet-v1-0)  

Communicates NHS 111 repeat caller document set which includes the NHS 111 Reports from repeat caller system along with the current NHS 111 Report to an Out of Hour recipient where the recipient is recommended to act upon information carried in the document set. Distribution envelope will carry multiple NHS 111 Reports as different payloads within the same envelope..

Note: Although the main message is not a CDA document, the imbedded CDA documents carried in this interaction have a message format that is constrained using HSCIC templates. This format is validated using a two level process. There are two sets of schema one for each level. These instances must also conform to the rules stated in the conformance definition indicated in the conformance format below.
